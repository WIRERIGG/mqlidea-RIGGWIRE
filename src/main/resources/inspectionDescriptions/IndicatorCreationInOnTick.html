Reports indicator handle creation (e.g., iMA(), iRSI(), iCustom()) inside OnTick() or OnCalculate() event handlers.

Creating indicator handles is an expensive operation that allocates internal buffers. Doing this every tick wastes resources and may leak handles if not properly released. Handles should be created once in OnInit() and stored for reuse.

Example (bad):

void OnTick() {
    int maHandle = iMA(_Symbol, PERIOD_H1, 14, 0, MODE_SMA, PRICE_CLOSE);
    double buf[];
    CopyBuffer(maHandle, 0, 0, 1, buf);
    IndicatorRelease(maHandle);
}

Example (good):

int maHandle;

int OnInit() {
    maHandle = iMA(_Symbol, PERIOD_H1, 14, 0, MODE_SMA, PRICE_CLOSE);
    return INIT_SUCCEEDED;
}

void OnTick() {
    double buf[];
    CopyBuffer(maHandle, 0, 0, 1, buf);
}

void OnDeinit(const int reason) {
    IndicatorRelease(maHandle);
}
