This inspection reports possible unintended fall-through in switch statements where a case block does not end with break or return before the next case. Accidental fall-through is a common source of bugs in MQL5 trading code.

Example of problematic code:

switch(signal) {
    case SIGNAL_BUY:
        OpenBuy();
        // Missing break — falls through to SIGNAL_SELL!
    case SIGNAL_SELL:
        OpenSell();
        break;
}

Add break to each case:

switch(signal) {
    case SIGNAL_BUY:
        OpenBuy();
        break;
    case SIGNAL_SELL:
        OpenSell();
        break;
}
