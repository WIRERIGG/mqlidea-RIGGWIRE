This inspection reports switch statements that do not contain a default case. Without a default branch, unhandled enum values or unexpected inputs will silently fall through, potentially causing incorrect behavior in trading logic.

Example of problematic code:

switch(orderType) {
    case ORDER_TYPE_BUY:
        // handle buy
        break;
    case ORDER_TYPE_SELL:
        // handle sell
        break;
    // Missing default case for other order types
}

Add a default case to handle unexpected values:

switch(orderType) {
    case ORDER_TYPE_BUY:
        // handle buy
        break;
    case ORDER_TYPE_SELL:
        // handle sell
        break;
    default:
        Print("Unexpected order type: ", orderType);
        break;
}
