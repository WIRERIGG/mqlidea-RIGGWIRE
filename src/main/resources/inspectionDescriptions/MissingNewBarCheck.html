Reports heavy computation (data copy operations) in OnTick() without a new bar check guard.

Functions like CopyRates(), CopyBuffer(), and CopyTicks() are expensive data operations. Running them on every tick wastes CPU when the strategy only needs to recalculate on new bars. Use a new bar detection pattern to skip redundant ticks.

Example (bad):

void OnTick() {
    double close[];
    CopyClose(_Symbol, PERIOD_H1, 0, 100, close);
    // Runs on every single tick
}

Example (good):

void OnTick() {
    static datetime lastBar = 0;
    datetime curBar = iTime(_Symbol, PERIOD_H1, 0);
    if (curBar == lastBar) return;
    lastBar = curBar;

    double close[];
    CopyClose(_Symbol, PERIOD_H1, 0, 100, close);
}
