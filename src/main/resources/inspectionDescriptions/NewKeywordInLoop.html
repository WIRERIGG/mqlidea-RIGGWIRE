Reports usage of the new keyword inside loop bodies (for, while, do).

Heap allocation inside a loop causes repeated memory allocation overhead per iteration. If objects are not properly deleted within the same iteration, this also leads to memory leaks. Pre-allocate objects before the loop or use object pools.

Example (bad):

for (int i = 0; i < total; i++) {
    COrder *order = new COrder();
    order.Process(i);
    delete order;
}

Example (good):

COrder order; // Stack allocation, reused each iteration
for (int i = 0; i < total; i++) {
    order.Process(i);
}
