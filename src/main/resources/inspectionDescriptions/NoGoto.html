This inspection reports usage of the goto statement in MQL5 code. The goto keyword creates unstructured control flow that is difficult to read, maintain, and reason about. Prefer structured alternatives such as loops, if/else, break, and continue.

Example of problematic code:

void OnTick() {
    goto retry;  // Warning: use structured control flow instead
retry:
    Print("retrying");
}

Preferred alternative:

void OnTick() {
    while(true) {
        Print("retrying");
        break;
    }
}
