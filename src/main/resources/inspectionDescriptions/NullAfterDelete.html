Reports delete statements that are not followed by setting the pointer to NULL.

After deleting an object, the pointer still holds the old address. Subsequent accidental use of that pointer causes undefined behavior (use-after-free). Setting the pointer to NULL immediately after delete prevents this class of bugs.

Example (bad):

delete m_indicator;
// m_indicator still points to freed memory

Example (good):

delete m_indicator;
m_indicator = NULL;
