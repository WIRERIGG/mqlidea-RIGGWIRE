This inspection reports a suspicious semicolon immediately after a control statement (if, for, or while). This pattern almost always indicates a logic error where the intended body is not actually controlled by the statement.

Example of problematic code:

if(OrderSelect(i, SELECT_BY_POS));   // Bug: semicolon makes the body empty
{
    OrderClose(OrderTicket(), OrderLots(), Ask, 3);
}

Fix by removing the stray semicolon:

if(OrderSelect(i, SELECT_BY_POS))
{
    OrderClose(OrderTicket(), OrderLots(), Ask, 3);
}
